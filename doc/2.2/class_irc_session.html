Detailed Description

Provides means to establish a connection to an IRC server.

Session management

Before opening a session, it must be first initialized with host, userName, nickName and realName.

The connection status may be queried at any time via isActive() and isConnected(). Changes in the connection status are informed via the following signals:

  • connecting() - The underlying socket has been connected, but the IRC connection is not yet fully established.
  • connected() - The IRC connection has been established, and the session is ready to start sending commands.
  • disconnected() - The session has been disconnected.

Receiving messages

Whenever a message is received from the server, the messageReceived() signal is emitted. Also message type specific signals are provided for convenience. See messageReceived() and IrcMessage and its subclasses for more details.

Sending commands

Sending commands to a server is most conveniently done by creating them via the various static IrcCommand::createXxx() methods and passing them to sendCommand(). Also sendData() is provided for more low-level access.

Example

IrcSession* session = new IrcSession(this);
connect(session, SIGNAL(connected()), this, SLOT(onConnected())); // ready to send commands
connect(session, SIGNAL(messageReceived(IrcMessage*)), this, SLOT(onMessageReceived(IrcMessage*)));
session->setHost("irc.server.com");
session->setUserName("me");
session->setNickName("myself");
session->setRealName("And I");
session->open();

void IrcSession::installMessageFilter ( IrcMessageFilter *  filter)

Installs a message filter on this session.

A message filter receives all messages that are sent to this session. The message filter receives events via its messageFilter() function. The messageFilter() function must return true if the message should be filtered, (i.e. stopped); otherwise it must return false.

If multiple message filters are installed on the same session, the filter that was installed last is activated first.

bool IrcSession::sendCommand ( IrcCommand *  command)

Sends a command to the server.

Note
If the command has a valid parent, it is an indication that the caller of this method is be responsible for freeing the command. If the command does not have a valid parent (like the commands created via various IrcCommand::createXxx() methods) the session will take ownership of the command and delete it once it has been sent. Thus, the command must have been allocated on the heap and it is not safe to access the command after it has been sent.

bool IrcSession::secure
readwrite

This property holds whether the socket is an SSL socket.

This property is provided for convenience. Calling

session->setSecure(true);

is equivalent to:

QSslSocket* socket = new QSslSocket(socket);
socket->setPeerVerifyMode(QSslSocket::QueryPeer);
session->setSocket(socket);
Note
IrcSession does not handle SSL errors, see QSslSocket::sslErrors() for more details on the subject.
