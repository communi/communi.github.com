      32                 :            : 
      33                 :            :     Since the list of supported commands and the exact syntax for each
      34                 :            :     command is application specific, IrcCommandParser does not provide
      35                 :            :     any built-in command syntaxes. It is left up to the applications
      36                 :            :     to introduce the supported commands and syntaxes.
      37                 :            :     IrcCommandParser supports the following command syntax markup:
      38                 :            : 
      39                 :            :     Syntax             | Example              | Description
      40                 :            :     -------------------|----------------------|------------
      41                 :            :     &lt;param&gt;      | &lt;target&gt;       | A required parameter.
      42                 :            :     (&lt;param&gt;)    | (&lt;key&gt;)        | An optional parameter.
      43                 :            :     &lt;param...&gt;   | &lt;message...&gt;   | A required parameter, multiple words accepted. (1)
      44                 :            :     (&lt;param...&gt;) | (&lt;message...&gt;) | An optional parameter, multiple words accepted. (1)

      46                 :            :     [param]            | [target]             | Inject the current target.
      47                 :            : 
      48                 :            :     -# Multi-word parameters are only supported in the last parameter position.
      49                 :            :     -# An optional channel parameter is filled up with the current channel when absent.
      50                 :            : 
      51                 :            :     The following example presents introducing some typical commands.
      52                 :            :     \code
      53                 :            :     IrcCommandParser* parser = new IrcCommandParser(this);
      54                 :            :     parser->addCommand(IrcCommand::Join, "JOIN <#channel> (<key>)");

      57                 :            :     parser->addCommand(IrcCommand::CtcpAction, "ME [target] <message...>");
      58                 :            :     parser->addCommand(IrcCommand::CtcpAction, "ACTION <target> <message...>");
      59                 :            :     \endcode
      60                 :            : 
      61                 :            :     \note The parameter names are insignificant, but descriptive
      62                 :            :     parameter names are recommended for the sake of readability.

      66                 :            :     Notice that commands are often context sensitive. While some command
      67                 :            :     may accept an optional parameter that is filled up with the current
      68                 :            :     target (channel/query) name when absent, another command may always
      69                 :            :     inject the current target name as a certain parameter. Therefore
      70                 :            :     IrcCommandParser must be kept up-to-date with the \ref target
      71                 :            :     "current target" and the \ref channels "list of channels".
      72                 :            :

      79                 :            :     \section triggers Command triggers
      80                 :            : 
      81                 :            :     IrcCommandParser serves as a generic parser for typical IRC commands.
      82                 :            :     It can be utilized for parsing commands from user input in GUI clients,
      83                 :            :     and from messages from other clients when implementing IRC bots.
      84                 :            : 
      85                 :            :     The command parsing behavior is controlled by setting up command
      86                 :            :     \ref triggers. Whilst a typical GUI client might use \c "/" as a command
      87                 :            :     trigger, an IRC bot might use \c "!" and the nick name of the bot. The
      88                 :            :     following snippet illustrates a typical GUI client usage.
      89                 :            : 
      90                 :            :     \code
